# 香港主要幹道交通阻塞時間預測模組 - 部署和接口調用指南

這份指南是專為 Dashboard 團隊及其他協作開發者編寫的。主要說明如何在本地快速運行這個基於 Spark 與 PyTorch 的後端預測模組，以及如何呼叫提供的 JSON 接口來獲取實時交通預測數據。

***

## 1. 快速啟動 (Local 環境)

本模組已經過封裝，並配置了自動降級機制（如果在沒有 GPU 的機器上運行，會自動切換為 CPU 推理，不會報錯）。在本地環境中運行非常簡單：

### 步驟 1: 安裝依賴環境

我們推薦使用 Conda 來管理環境。
請在終端機 (Terminal) 中執行以下命令，創建環境並安裝必要的套件：

```bash
# 建立名為 hk_traffic 的虛擬環境 (Python 3.10)
conda create -y -n hk_traffic python=3.10

# 啟動環境
conda activate hk_traffic

# 安裝 Java (PySpark 必須的依賴)
conda install -y -c conda-forge openjdk=17

# 安裝 PyTorch 及其他相依套件
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
pip install -r requirements.txt
```

### 步驟 2: 啟動後端服務

確保你位於專案根目錄 (`hk_traffic_prediction`) 下，然後執行我們準備好的一鍵啟動腳本：

```bash
# 確保你已經 activate 了 hk_traffic 環境
bash run_local.sh
```

**啟動成功的標誌：**
你會在終端機中看到如下日誌：

```text
Fetching realtime data...
Running Spark ETL...
Running Predictor...
Updated predictions for 3845 segments.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

這代表後端已經成功從香港政府的 Open Data 平台拉取了最新的 XML 資料，並透過 Spark 和 PyTorch 完成了第一次預測，服務已經在 `8000` 端口上準備就緒。

> **💡 提示**：後台已經設定了定時任務，每 5 分鐘會自動重複上述流程（拉取資料 -> Spark ETL -> PyTorch 推理），確保緩存中的預測結果永遠是最新的。

***

## 2. 瀏覽器測試

當服務啟動後，你可以直接在瀏覽器中輸入以下網址進行簡單的連通性測試：

1. **檢查服務狀態**：
   打開 `http://127.0.0.1:8000/`
   你應該會看到包含所有可用端點的 JSON 回應：
   ```json
   {"message":"Traffic Prediction API is running","endpoints":["/predict?segment_id=XXXX","/predictions"],"status":"Success"}
   ```
2. **查看所有預測結果**：
   打開 `http://127.0.0.1:8000/predictions`
   你將會看到一個包含幾千個路段預測資料的龐大 JSON 檔案。

## 3. 接口調用文件 (API Documentation)

Dashboard 同學可以透過以下 REST API 獲取最新的路況預測資料，並整合到前端地圖或側邊欄中。

### 接口一：批量獲取所有主要路段預測 (推薦 Dashboard 使用)

**GET** `/predictions`

- **用途**：一次性取得所有路段的最新預測資料。適合用於地圖全局渲染（例如在地圖上將所有 `predicted_congestion_minutes > 10` 的路段標示為紅色）。
- **請求參數**：無
- **回傳格式**：`application/json`
- **回傳範例**：
  ```json
  {
    "count": 3845,
    "predictions": {
      "105500": 0.01,
      "105543": 12.5,
      "106257": 0.0,
      "278128": 25.3
    }
  }
  ```
  *(註：`predictions`* *內的 Key 為* *`segment_id`* *(字串)，Value 為預測的未來 30 分鐘阻塞時間 (分鐘)。)*
- **Dashboard 對接建議 (JavaScript)**：
  由於後端每 5 分鐘才會更新一次資料，建議前端使用 `setInterval` 每 5 分鐘（300000 毫秒）輪詢一次這個接口：
  ```javascript
  async function fetchTrafficPredictions() {
      try {
          const response = await fetch('http://127.0.0.1:8000/predictions');
          const data = await response.json();
          console.log(`成功獲取 ${data.count} 條路段預測`);
          // 更新你的地圖狀態...
          updateMap(data.predictions);
      } catch (error) {
          console.error("無法取得預測資料", error);
      }
  }

  // 初始加載
  fetchTrafficPredictions();
  // 每 5 分鐘更新一次
  setInterval(fetchTrafficPredictions, 5 * 60 * 1000);
  ```

### 接口二：單一路段詳細查詢

**GET** `/predict`

- **用途**：查詢特定路段的預測資料。適合用於使用者點擊地圖上某個路段時，彈出的詳細資訊視窗 (Tooltip/Popup)。
- **請求參數**：
  - `segment_id` (整數，必填)：你要查詢的路段 ID。
- **請求範例**：`http://127.0.0.1:8000/predict?segment_id=105500`
- **回傳格式**：`application/json`
- **回傳範例**：
  ```json
  {
    "segment_id": 105500,
    "predicted_congestion_minutes": 12.5,
    "status": "Success"
  }
  ```
  *(若查詢的路段不存在或沒有預測資料，`predicted_congestion_minutes`* *將會回傳* *`null`，且* *`status`* *為* *`"Not found or no data"`)*
