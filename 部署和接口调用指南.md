# 香港主要幹道交通阻塞時間預測模組 - 部署和接口調用指南

這份指南是專為 Dashboard 團隊及其他協作開發者編寫的。主要說明如何在本地快速運行這個基於 Spark 與 PyTorch 的後端預測模組，以及如何呼叫提供的 JSON 接口來獲取實時交通預測數據。

***

## 1. 快速啟動 (Local 環境)

本模組已經過封裝，並配置了自動降級機制（如果在沒有 GPU 的機器上運行，會自動切換為 CPU 推理，不會報錯）。在本地環境中運行非常簡單：

### 步驟 1: 安裝依賴環境

我們推薦使用 Conda 來管理環境。
請在終端機 (Terminal) 中執行以下命令，創建環境並安裝必要的套件：

```bash
# 建立名為 hk_traffic 的虛擬環境 (Python 3.10)
conda create -y -n hk_traffic python=3.10

# 啟動環境
conda activate hk_traffic

# 安裝 Java (PySpark 必須的依賴)
conda install -y -c conda-forge openjdk=17

# 安裝 PyTorch 及其他相依套件
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
pip install -r requirements.txt
# MacOS 用户由于环境解析问题，建议单独使用 pip 安装空间地理库：
pip install geopandas fiona pyogrio shapely
```

### 步驟 2: 預處理空間路網數據 (僅需一次)

地圖功能依賴於將香港政府的官方 FGDB 格式路網處理為輕量級的 GeoJSON。
確保你已經下載了 `RdNet_IRNP.gdb.zip` 並解壓到了 `data/road_network/raw/` 資料夾下。

```bash
# 提取中心線、過濾不需要的道路、簡化幾何並生成 GeoJSON
python src/visualization/prepare_road_network.py
```
這將在 `data/road_network/processed/` 下生成 `road_network.geojson` 檔案。

### 步驟 3: 啟動後端服務

確保你位於專案根目錄 (`hk_traffic_prediction`) 下，然後執行我們準備好的一鍵啟動腳本。你可以通过参数决定 MTR 模块是否使用真实数据：

```bash
# 默认模式（MTR 预测返回 Mock 假数据，适合开发与测试）
bash run_local.sh

# 真实模式（MTR 预测将读取最新采集的 JSON 数据进行推理）
bash run_local.sh --real
```
*(注意：在真实模式下，你必须确保 `data_logger.py` 正在后台运行，否则 API 会找不到最新数据。)*

### 步骤 4: 启动 MTR 数据采集 (仅真实模式需要)

由于 MTR 的预测需要高频采集列车信息，数据抓取脚本是**独立于后端服务**运行的。如果你想在真实模式下运行，请在另一个终端中启动 Logger：

```bash
conda activate hk_traffic
bash run_mtr_logger.sh
```
这将在后台每 30 秒抓取一次全港铁线路的实时状态，并存入 `data/historical/mtr_nexttrain/raw/` 供模型推理和训练使用。

### 5. 关闭与重启服务

- **关闭服务 (Shutdown)**: 
  如果你是通过 `bash run_local.sh` 在终端启动的，可以直接在终端窗口按下 `Ctrl + C` 来停止 FastAPI 和 Spark 的运行。
  如果你是通过 Docker 启动的，可以使用 `docker stop <container_id>` 命令。

- **重启服务 (Restart)**:
  只要你修改了 `src/api/main.py` 或相关文件，你可以先按下 `Ctrl + C` 停止，然后再次运行 `bash run_local.sh` 即可重启服务。

## 3. 浏览器测试

当服务启动后，你可以直接在浏览器中输入以下网址进行简单的连通性测试：

1. **查看实时交通交互地图**（新增功能）：
   打开 `http://127.0.0.1:8000/map`
   你将会看到一个香港交通路网的互动地图。该地图会每 30 秒通过智能 Diff 策略从后端拉取一次数据，根据预测的拥堵时长对路段进行着色（绿、黄、橙、红）。

2. **检查服务状态**：
   打开 `http://127.0.0.1:8000/`
   你应该会看到包含所有可用端点的 JSON 回应。

3. **查看所有预测结果**：
   打开 `http://127.0.0.1:8000/predictions`
   你将会看到一个包含几千个路段预测资料和 `last_update` 时间戳的 JSON 文件。

## 4. 接口调用文件 (API Documentation)

Dashboard 同学可以透过以下 REST API 获取最新的路况预测资料，并整合到前端地图或侧边栏中。

### 接口一：获取地图前端配置 (新增)

**GET** `/map_config`

- **用途**：获取地图渲染的核心配置，避免前端硬编码。
- **回传范例**：
  ```json
  {
    "map_center": [22.3193, 114.1694],
    "color_thresholds": [0, 5, 20],
    "prediction_api_endpoint": "/predictions"
  }
  ```

### 接口二：批量获取所有主要路段预测 (推荐 Dashboard 使用)

**GET** `/predictions`

- **用途**：一次性取得所有路段的最新預測資料。適合用於地圖全局渲染（例如在地圖上將所有 `predicted_congestion_minutes > 10` 的路段標示為紅色）。
- **請求參數**：無
- **回傳格式**：`application/json`
- **回傳範例**：
  ```json
  {
    "count": 3845,
    "last_update": "2026-04-18 11:02:48",
    "predictions": {
      "105500": 0.01,
      "105543": 12.5,
      "106257": 0.0,
      "278128": 25.3
    }
  }
  ```
  *(註：`predictions`* *內的 Key 為* *`segment_id`* *(字串)，Value 為預測的未來 30 分鐘阻塞時間 (分鐘)。)*
- **Dashboard 对接建议 (JavaScript)**：
  由于后端每 5 分钟才会更新一次资料，建议前端使用 `setInterval` 每 30 秒轮询一次这个接口，并结合数据中的 `last_update` 字段进行对比，避免无意义的页面重绘渲染：
  ```javascript
  // 全局变量保存最后一次更新的时间戳
  let lastBackendUpdateTime = "";

  async function fetchTrafficPredictions() {
      try {
          const response = await fetch('http://127.0.0.1:8000/predictions');
          const data = await response.json();
          
          // 智能更新策略：如果后端时间戳没有变化，就不重新渲染地图
          if (data.last_update && data.last_update === lastBackendUpdateTime) {
              console.log("后端预测数据尚未更新，跳过渲染。");
              return;
          }
          
          lastBackendUpdateTime = data.last_update || "";
          console.log(`成功获取 ${data.count} 条路段预测，数据生成时间: ${lastBackendUpdateTime}`);
          
          // 更新你的地图状态...
          updateMap(data.predictions);
      } catch (error) {
          console.error("无法取得预测资料", error);
      }
  }

  // 初始加载
  fetchTrafficPredictions();
  // 建议每 30 秒轮询一次，结合上述 Diff 策略，兼顾实时性和前端渲染性能
  setInterval(fetchTrafficPredictions, 30 * 1000);
  ```

### 接口三：单一路段详细查询

**GET** `/predict`

- **用途**：查詢特定路段的預測資料。適合用於使用者點擊地圖上某個路段時，彈出的詳細資訊視窗 (Tooltip/Popup)。
- **請求參數**：
  - `segment_id` (整數，必填)：你要查詢的路段 ID。
- **請求範例**：`http://127.0.0.1:8000/predict?segment_id=105500`
- **回傳格式**：`application/json`
- **回傳範例**：
  ```json
  {
    "segment_id": 105500,
    "predicted_congestion_minutes": 12.5,
    "status": "Success"
  }
  ```
  *(若查询的路段不存在或没有预测资料，`predicted_congestion_minutes`* *将会回传* *`null`，且* *`status`* *为* *`"Not found or no data"`)*

### 迁移至云端 (Cloud Deployment)

整个专案采用配置驱动 (`config-driven`)，前端代码没有任何硬编码。
1. **修改配置**：你只需要修改 `config/cloud.yaml` 中的 `master` 属性指向你的 Spark 集群 IP 即可。
2. **构建 Docker 镜像**：我们已经更新了 `Dockerfile`，将前端资源和最新的地理空间依赖打包了进去：
   ```bash
   docker build -t hk-traffic-prediction .
   ```
3. **启动容器**：
   ```bash
   docker run -d -p 8000:8000 --name traffic_api hk-traffic-prediction
   ```
   *(Docker 中已经将环境变量 `ENV=cloud` 预设好了，服务会自动加载 `cloud.yaml`。)*
