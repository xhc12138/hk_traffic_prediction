# 专营巴士线网监控与实时预报子模块 (Bus Network Module)

本模块是香港交通预测系统的重要扩展部分，专注于提供香港专营巴士（以九巴 KMB 为主）的线网空间可视化与车辆实时到站时间 (ETA) 监控。

---

## 1. 模块核心功能

### 1.1 静态线网与站点空间映射 (Spark RDD)
香港政府开放数据平台提供的巴士数据通常将“路线 (Route)”、“站点列表 (Stops)”和“站点坐标 (Geospatial Data)”分散在不同的庞大 JSON/CSV 文件中。
本模块利用 **Spark RDD 和 DataFrame** 技术，实现了高效的大数据空间联表操作：
- **扁平化处理 (Explode)**：将嵌套的路线-站点序列数组进行展开。
- **分布式关联 (Join)**：将路线字典、站点序列与站点物理经纬度进行高效 Join，生成每条路线的完整物理轨迹。
- **前端轻量化**：后端处理后，仅输出前端渲染所需的最简 JSON 格式，避免前端加载数 GB 的原始数据。

### 1.2 实时到站时间 (ETA) 与虚拟车辆推算
- **官方 API 对接**：系统实时对接九巴官方的 ETA 开放 API (`https://data.etabus.gov.hk/`)。
- **懒加载与预加载结合**：
  - **预加载 (Pre-fetch)**：针对常用于演示的核心路线（如 Route 1 和 Route 10），后端 `APScheduler` 每 30 秒自动在后台轮询更新缓存，前端请求实现 **0 延迟秒开**。
  - **懒加载 (Lazy Load)**：对于其余数百条路线，仅在用户于前端点击时才发起异步请求，最大限度节省网络带宽与服务器资源。
- **虚拟车辆插值动画**：由于官方 API 不提供车辆绝对 GPS 坐标，仅提供“距离前方站点的预计分钟数”，本模块在前端实现了一套**空间插值算法 (Spatial Interpolation)**。算法基于前后站点的物理坐标和 ETA 剩余时间，动态推算并渲染出“行驶在路上”的 🚌 巴士图标。

## 2. 文件结构与职责

该子模块的代码主要分布在以下目录：

- `data/historical/bus/`：存放从政府平台获取的原始巴士路线与站点数据（`route_kmb.json`, `stop_kmb.json` 等）。
- `src/bus/inference/spark_etl_bus.py`：核心大数据处理脚本。负责读取原始 JSON 文件，执行 Spark DataFrame 的 `explode` 和 `join` 操作，输出结构化的路线轨迹数据。
- `src/api/main.py`：
  - 增加后台定时任务 `update_bus_data()` 更新静态线网。
  - 增加后台定时任务 `update_prefetched_bus_eta()` 预加载热门路线 ETA。
  - 提供 RESTful 接口 `/bus/routes` 和 `/bus/eta/{route}`。
- `frontend/map.js`：
  - 包含 `loadBusNetwork()`、`selectBusRoute()` 等前端渲染逻辑。
  - 实现 Leaflet 侧边栏联动、Polyline 路线绘制、CircleMarker 站点渲染以及动态巴士图标推算。

## 3. 接口调用指南

### GET `/bus/routes`
- **功能**：获取全港巴士路线的基础信息与途径站点物理坐标数组。
- **返回示例**：
  ```json
  {
      "count": 400,
      "last_update": "2026-04-24 15:00:00",
      "routes": [
          {
              "route": "1",
              "bound": "O",
              "stops": [
                  {"seq": 1, "lat": "22.345", "long": "114.192", "name_tc": "竹園邨總站"},
                  {"seq": 2, "lat": "22.344", "long": "114.190", "name_tc": "天虹小學"}
              ]
          }
      ]
  }
  ```

### GET `/bus/eta/{route}`
- **功能**：获取指定路线的实时 ETA 数据。
- **参数**：`route` - 路线编号（如 `1`, `1A`, `261`）。不区分大小写，自动去除前导零。
- **返回示例**：
  ```json
  {
      "type": "StopETA",
      "data": [
          {
              "route": "1",
              "dir": "O",
              "seq": 1,
              "eta": "2026-04-24T15:20:00+08:00"
          }
      ]
  }
  ```

## 4. 前端交互体验
1. **纯净视图**：点击右上角切换至 `Bus Network` 系统时，地图会自动清理道路拥堵和港铁警报信息。
2. **侧边栏联动**：左侧显示路线手风琴列表，点击某一路线时，地图会自动清空其余路线，仅高亮显示当前选中的路线并自动缩放至最佳视野 (`fitBounds`)。
3. **沉浸式动态 ETA**：地图上的车站圆点在即将有车到站时会闪烁黄光，同时路面上会出现动态推算的 🚌 巴士图标，悬停即可查看详细到站倒计时。
